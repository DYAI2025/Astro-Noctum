### ElevenLabs Webhook Tool Schema (Current as of 2026)

The top-level structure must be:
```json
{
  "type": "webhook",
  "name": "tool_name",
  "description": "...",
  "api_schema": {
    "url": "https://...",
    "method": "GET" | "POST",
    "path_params_schema": [...],
    "query_params_schema": [...],
    "request_body_schema": [...],
    "request_headers": [...]
  }
}
```

**CRITICAL rules (learned from past failures):**

- `type` MUST be at top level (NOT inside a `tool_config` wrapper)
- Valid `type` values: `webhook` | `client` | `system` | `api_integration_webhook`
- `request_headers` items MUST have:
  - `key`: header name (e.g. "Authorization")
  - `value`: header value (e.g. "Bearer {{SECRET}}")
  - `type`: one of `value` | `secret` | `dynamic_variable`
  - If `type` is `secret`: `secret_id` is REQUIRED
- `path_params_schema` items need: `id`, `type`, `description`, and optionally `dynamic_variable` or `constant_value`
- `request_body_schema` items need: `id`, `type`, `description`, and optionally `dynamic_variable`
- All string values must be properly escaped (no raw newlines, tabs, or control characters)
- The JSON must be parseable as a single-line string (for paste into UI)

### Common Errors and Fixes

| Error | Fix |
|-------|-----|
| `type: Invalid discriminator value` | Move `type: "webhook"` to top level, remove any `tool_config` wrapper |
| `request_headers.N.type: Invalid discriminator value` | Add `"type": "value"` or `"type": "secret"` to each header |
| `request_headers.N.name: Required` | Use `"key"` not `"name"` for the header field name |
| `request_headers.N.secret_id: Header secret is required` | Add `"secret_id": "your-secret-name"` when type is "secret" |
| `Bad control character in string literal` | Escape newlines, tabs, quotes in description strings |

### Guardrails

- Never remove fields — only fix/add what the error requires
- Keep the existing description, URL, and parameter schemas intact
- If unsure about a field, fetch the latest docs: https://elevenlabs.io/docs/agents-platform/customization/tools/server-tools
- After fixing, pretty-print the JSON for readability
