# Dashboard Tiles Redesign

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Redesign 7 dashboard tile types for maximum personalization: fix squished/generic icons, use dedicated zodiac & coin art, replace generic icons with Chinese characters, expand Gemini to generate personalized per-tile text + house tooltips, and add collapsible "more" sections.

**Architecture:** One extended Gemini prompt generates all personalized text in a single call (interpretation + tile expansions + house tooltips). Asset pipeline converts new PNGs to WebP. Dashboard components consume the expanded response via a structured JSON contract.

**Tech Stack:** React 19, Vite, Tailwind v4, Framer Motion, Gemini 2.0 Flash, sharp (one-shot asset conversion)

**Design Principle:** Maximum personalization — every tile should feel like it was written for THIS user's chart.

## Gemini Prompt Expansion

**Current:** Single prompt → 5 paragraphs of general interpretation (400-500 words).

**New:** Single prompt → structured JSON response containing:

```json
{
  "interpretation": "## Dein Bazodiac Fusion-Blueprint\n\n...(5 paragraphs, 400-500 words)...",
  "tiles": {
    "sun": "2-3 Sätze personalisierte Sonnenzeichen-Deutung...",
    "moon": "2-3 Sätze personalisierte Mondzeichen-Deutung...",
    "yearAnimal": "2-3 Sätze über Tier+Element Kombination...",
    "dominantWuXing": "2-3 Sätze personalisierte Element-Deutung...",
    "dayMaster": "2-3 Sätze personalisierte Tagesmeister-Deutung..."
  },
  "houses": {
    "1": "2-3 Sätze: Widder in deinem 1. Haus bedeutet...",
    "2": "2-3 Sätze: Stier in deinem 2. Haus bedeutet...",
    ...
    "12": "..."
  }
}
```

**Constraints for Gemini:**
- Only use data from the actual chart (no hallucinations)
- Each tile text must reference the specific sign/element/animal
- House texts must reference the specific sign IN that house
- Language = active `lang`, address as "du" (de) or "you" (en)
- Temperature stays at 0.75
- Estimated total: ~1200-1500 words (vs. current ~500). Still within Gemini Flash limits.

**Fallback:** If Gemini returns plain text (not JSON), fall back to current behavior — interpretation only, no tile expansions, no house tooltips.

## Collapsible "More" Component

A simple reusable component for tile expansions:

```tsx
function ExpandableText({ text, labelExpand, labelCollapse }: Props) {
  const [open, setOpen] = useState(false);
  if (!text) return null;
  return (
    <div className="mt-3 border-t border-[#8B6914]/10 pt-3">
      <button onClick={() => setOpen(!open)} className="text-[10px] text-[#8B6914] uppercase tracking-wider">
        {open ? labelCollapse : labelExpand}
      </button>
      {open && <p className="text-xs text-[#1E2A3A]/55 leading-relaxed mt-2">{text}</p>}
    </div>
  );
}
```

Placed inside each tile card, below the static description. Only renders when Gemini text is available.

## Data Flow Change

```
Current:  Gemini → string (markdown) → interpretation state → Dashboard
New:      Gemini → JSON string → parse → { interpretation, tiles, houses } → Dashboard
```

`App.tsx` stores the parsed response. `AppLayoutContext` expands to include `tileTexts` and `houseTexts`. Dashboard tiles read from context.
