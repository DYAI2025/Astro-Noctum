ARCHITEKTUR-ENTSCHEIDUNG
Bevor Tag 1 startet, eine strategische Weichenstellung:
Monetarisierungs-Architektur
Du hast zwei Wege. Ich empfehle die Kombination:
Weg A: AdSense + Traffic (langsam, passiv, skaliert)
AdSense ist schon eingebunden. Aber AdSense braucht Traffic. Traffic braucht SEO. SEO braucht Content-Seiten. Eine SPA wird von Google schlecht indexiert. Du brauchst entweder SSR (teuer an Umbau) oder eine separate Content-Schicht (Astro/Next.js Landing Pages, die auf die App verlinken).
Zeitrahmen bis Revenue: 6–12 Wochen (bis Google indexiert und Traffic kommt)
Revenue-Potenzial: 50–500€/Monat bei 10k–50k Visits (Nische Astrologie)
Weg B: Freemium + Stripe (schnell, direkt, begrenzt)
Basic Reading gratis (Sonnenzeichen + Basis-Bazi). Premium Reading hinter Paywall: Full Fusion, detaillierte Interpretation, Voice Agent, PDF-Export. Einmal-Zahlung oder kleines Abo.
Zeitrahmen bis Revenue: Sofort nach Launch (wenn Nutzer kommen)
Revenue-Potenzial: 5–29€ pro Premium Reading, skaliert mit Nutzern
EMPFEHLUNG: Beides. Freemium für sofortigen Revenue-Pfad. Content-Seiten für langfristigen Traffic. AdSense als Bonus auf den Content-Seiten.

Fokus: Payment-Flow
Aufgaben:

Start mit Stufe 1. Nicht overengineer-en. EIN Preis, EIN Button, EIN Checkout.
Technische Umsetzung:
☐  Stripe Checkout Session API in server.mjs: POST /api/checkout
☐  Webhook-Endpoint: POST /api/webhook/stripe → setzt profiles.tier = 'premium'
☐  Frontend: "Upgrade to Premium" Button → redirectet zu Stripe Checkout → nach Zahlung zurück zur App
☐  Supabase profiles.tier nach erfolgreicher Zahlung updaten
// server.mjs - Minimal Stripe Integration
import Stripe from 'stripe';
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);
app.post('/api/checkout', async (req, res) => {
  const session = await stripe.checkout.sessions.create({
    line_items: [{ price: PRICE_ID, quantity: 1 }],
    mode: 'payment',
    success_url: `${DOMAIN}?upgrade=success`,
    metadata: { userId: req.body.userId }
  });
  res.json({ url: session.url });
});
→ DONE-GATE: Test-Zahlung in Stripe Test-Mode funktioniert. User wird nach Zahlung als Premium markiert. Premium-Content ist sichtbar.
